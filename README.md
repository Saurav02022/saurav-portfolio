# Saurav Kumar — Portfolio

Personal portfolio of **Saurav Kumar**, a full-stack + LLM engineer. Single-page, light, typography-led — built with Next.js (App Router), TypeScript, and Tailwind CSS v4, and deployed on Vercel.

**Live:** [saurav02022-portfolio.vercel.app](https://saurav02022-portfolio.vercel.app)

## Highlights

- **Data-driven content** — every string on the site lives in one typed file ([lib/portfolio-data.tsx](lib/portfolio-data.tsx)); components only render it, so copy changes never touch component code.
- **Six case studies**, each with its own architecture figure.
- **SEO** — Next.js Metadata API, generated `sitemap.xml` / `robots.txt`, JSON-LD, and OG/Twitter cards rendered from design tokens at build.
- **Accessible** — skip link, `:focus-visible` rings, `prefers-reduced-motion` support, and WCAG AA body text.

## Tech stack

- **Next.js 16** (App Router) · **React 19**
- **TypeScript**
- **Tailwind CSS v4** — design tokens as CSS variables in [app/globals.css](app/globals.css); no separate config file
- **next/font/google** — Syne, Instrument Sans, Space Mono
- **Vercel** — hosting and analytics

The visual system and its rules are documented in [design.md](design.md).

## Editing content

All copy lives in [lib/portfolio-data.tsx](lib/portfolio-data.tsx) (typed by [lib/types.ts](lib/types.ts)) — cover, work, experience, about, toolkit, contact, socials, and the résumé URL. Content changes never require component edits.

To serve the résumé locally, drop `Saurav-Kumar-Resume.pdf` into `public/` and point `RESUME_URL` at `/Saurav-Kumar-Resume.pdf`.

## Development

```bash
npm install
npm run dev     # http://localhost:3000
npm run build   # production build
npm run lint
```

## SEO

- Canonical URL from `NEXT_PUBLIC_SITE_URL` (defaults in [lib/site-config.ts](lib/site-config.ts))
- `robots.txt` and `sitemap.xml` generated by [app/robots.ts](app/robots.ts) and [app/sitemap.ts](app/sitemap.ts)
- JSON-LD (Person, WebSite, and one SoftwareSourceCode per project) in [components/structured-data.tsx](components/structured-data.tsx)
- OG/Twitter share cards and favicons generated at build from design tokens ([app/opengraph-image.tsx](app/opengraph-image.tsx), [app/twitter-image.tsx](app/twitter-image.tsx), [app/icon.tsx](app/icon.tsx))
- Google verification via `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION`; visitor stats via Vercel Analytics
